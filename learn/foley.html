How Foley Protocol Works

The Classification Engine

The classifier looks at your prompt and extracts signals. How long is it? Are there code blocks? Is it asking for analysis or generation? Does it mention design, architecture, creative work?

It then maps those signals to a tier. Here's the simplified logic:

function classifyComplexity(input, context) { const text = input.toLowerCase(); // Check for creative markers if (CREATIVE_MARKERS.some(m => text.includes(m))) { return 'CREATIVE'; // write, voice, tone, persona } // Check for design/architecture markers if (DESIGN_MARKERS.some(m => text.includes(m))) { return 'COMPLEX'; // design, architecture, implement } // Check for reasoning complexity if (reasoningSteps >= 5 || codeLines > 200) { return 'COMPLEX'; } // TRIVIAL: short, no complex markers if (textLength < 500 && !hasComplexMarker && simpleTools) { return 'TRIVIAL'; } return 'STANDARD'; // default: moderate reasoning }

The real Foley implementation is 591 lines of battle-tested code with throttling, retry logic, 429 detection, and cross-provider failover. But the core idea is simple: read the prompt, pick the right tier, route accordingly.

Automatic Failover: Never Miss a Query

If Google's rate limit kicks in, Foley detects the 429 and immediately routes to the next eligible provider. If a request times out, it retries with increasing delays (1s → 5s → 15s → 60s). The system never gives up unless everything is genuinely down.

This is what separates a production routing system from a simple API call. Foley is built for real workloads.
